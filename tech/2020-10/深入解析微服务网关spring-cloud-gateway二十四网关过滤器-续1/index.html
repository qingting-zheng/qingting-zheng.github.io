网关过滤器工厂接口

路由过滤器允许以某种方式修改传入的HTTP请求或传出的HTTP响应。 路由过滤器的作用域是一个特定的路由。Spring Cloud Gateway包含许多内置的GatewayFilter工厂。首先看一下GatewayFilterFactory接口中定义的方法。

public interface GatewayFilterFactory<C> extends ShortcutConfigurable, Configurable<C> {
	default GatewayFilter apply(Consumer<C> consumer) {
		C config = newConfig();
		consumer.accept(config);
		return apply(config);
	}
	default String name() {
		return NameUtils.normalizeFilterName(getClass());
	}
}

这种类型的接口实在是熟悉，default类型的方法name()用于对过滤器标准化命名，#apply方法用于定义具体的过滤操作，泛型C是定义在各个实现类中的配置，将相应的配置进行应用。如下图所示为GatewayFilterFactory的类图。

抽象类AbstractGatewayFilterFactory实现了GatewayFilterFactory接口，和之前讲断言工厂时的抽象类功能一样，为了兼容参数为元组类型的情况。另一个抽象类AbstractNameValueGatewayFilterFactory继承自AbstractGatewayFilterFactory，从起命名就可以看书，该抽象类用于提供通用的方法给键值对类型的网关过滤器。

public abstract class AbstractNameValueGatewayFilterFactory extends AbstractGatewayFilterFactory<AbstractNameValueGatewayFilterFactory.NameValueConfig> {
	public AbstractNameValueGatewayFilterFactory() {
		super(NameValueConfig.class);
	}
	public List<String> shortcutFieldOrder() {
        return Arrays.asList(GatewayFilter.NAME_KEY, GatewayFilter.VALUE_KEY);
    }
	@Validated
	public static class NameValueConfig {
		@NotEmpty
		protected String name;
		@NotEmpty
		protected String value;
		...
	}
}

从如上的实现来看，该抽象类为键值对类型的网关过滤器提供了通用的配置类，即之前提到的泛型C。除此之外，返回有关参数数量和解析顺序的列表。网关过滤器有近二十个实现类，总得说来可以分为七类：Header、Parameter、Path、Status、Redirect跳转、Hystrix熔断和RateLimiter限流。
